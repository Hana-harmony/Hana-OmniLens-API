# ADR-002: 단일 노드 운영에서 Kubernetes를 도입하지 않는다

- 상태: 승인
- 일자: 2026-07-15

## 결정

OCI와 AWS의 현재 단일 노드 서비스는 Docker Compose와 systemd로 배포한다. 관측성도 호스트별 독립 Compose 스택으로 운영한다.

## 근거

단일 노드 Kubernetes는 노드 장애를 복구하지 못한다. 반면 제어 평면, CNI, Ingress, 영속 볼륨과 업그레이드 운영 부담이 추가된다. 현재 자원에서는 애플리케이션과 데이터베이스에 자원을 우선 배분하는 편이 장애 가능성과 복구 시간을 줄인다.

## 재검토 조건

서로 다른 장애 도메인의 노드가 최소 3개 확보되고, 서비스별 독립 확장·자동 스케줄링·정책 기반 배포가 실제 요구될 때 관리형 Kubernetes를 검토한다.
